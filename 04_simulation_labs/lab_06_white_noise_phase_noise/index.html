Lab 06 — 白噪 → 1/f² 相位雜訊

這個 lab 把整條因果鏈「白噪 → ISF 加權 → 積分 → 相位」用最乾淨的數值方式跑一次， 讓你親眼看到一條平的（white，白色，與頻率無關）電流雜訊功率譜，怎麼被振盪器 這個「相位積分器」染成 1/f21/f^21/f2（每十倍頻 −20-20−20 dB）的相位雜訊。

物理直覺（先講結論）：振盪器的相位沒有恢復力（restoring force，把擾動拉回的力）， 所以每一次 noise 踢出的相位步階都永久累加——這就是一個積分器（integrator， 對輸入做時間積分的系統，infinite memory）。積分器的轉移函數量值是 1/(2πf)1/(2\pi f)1/(2πf)， 對 PSD 是 1/(2πf)21/(2\pi f)^21/(2πf)2。輸入是平的（白噪），輸出就被乘上 1/f21/f^21/f2。白進、1/f21/f^21/f2 出， 不是因為雜訊源是 1/f21/f^21/f2，而是因為相位是被積分出來的。

1. 教學目標​

  • 用模擬驗證 [P1] 的招牌結果：白色電流雜訊在振盪器中變成 1/f21/f^21/f2 相位雜訊。
  • 看懂 −20-20−20 dB/decade 斜率從哪裡來（來自積分器，不是來自噪聲源）。
  • 量化驗證理論預測式 Sϕ(f)=Γrms2Si/(qmax2(2πf)2)S_\phi(f)=\Gamma_{rms}^2 S_i/(q_{max}^2(2\pi f)^2)Sϕ​(f)=Γrms2​Si​/(qmax2​(2πf)2)。
  • 理解著名的 factor-of-2（2 倍因子）爭議：時域乾淨推導與 [P1] Eq.(21) 差一個 2 倍， 但 scaling 與斜率完全不受影響。

2. 數學模型​

相位響應就是 LTV（linear time-variant，線性時變）卷積式 [P1] Eq.(11), p.182：

ϕ(t)=1qmax∫−∞tΓ(ω0τ) in(τ) dτ\phi(t)=\frac{1}{q_{max}}\int_{-\infty}^{t}\Gamma(\omega_0\tau)\,i_n(\tau)\,d\tauϕ(t)=qmax​1​∫−∞t​Γ(ω0​τ)in​(τ)dτ

把它拆成三個 block：先用 ISF 對 noise 做相位加權，再做積分。輸入是白色電流雜訊， 其單邊 PSD（power spectral density，功率譜密度）為常數 SiS_iSi​（A2/A^2/A2/Hz）。

對白色輸入做積分，輸出相位 PSD 的乾淨時域結果是

Sϕ(f)=Γrms2 Siqmax2 (2πf)2.S_\phi(f)=\frac{\Gamma_{rms}^2\,S_i}{q_{max}^2\,(2\pi f)^2}.Sϕ​(f)=qmax2​(2πf)2Γrms2​Si​​.
  • 怎麼來的：ISF 把白噪加權後，平均功率被 Γrms2\Gamma_{rms}^2Γrms2​（ISF 的 rms 平方， 見 [P1] Eq.(20)）縮放，qmax2q_{max}^2qmax2​ 在分母（normalize）；接著積分器在頻域是 1/(j2πf)1/(j2\pi f)1/(j2πf)，對 PSD 是 1/(2πf)21/(2\pi f)^21/(2πf)2。
  • Dimension check：Γrms2\Gamma_{rms}^2Γrms2​ 無因次、SiS_iSi​ 是 A2/A^2/A2/Hz、qmax2q_{max}^2qmax2​ 是 C2C^2C2、 (2πf)2(2\pi f)^2(2πf)2 是 (rad/s)2=s−2(\text{rad/s})^2=\text{s}^{-2}(rad/s)2=s−2。整體 =A2/HzC2⋅s−2=A2 s2C2⋅1Hz=\dfrac{A^2/\text{Hz}}{C^2\cdot\text{s}^{-2}}=\dfrac{A^2\,\text{s}^2}{C^2}\cdot\dfrac{1}{\text{Hz}}=C2⋅s−2A2/Hz​=C2A2s2​⋅Hz1​。 因為 C=A⋅sC=A\cdot sC=A⋅s，A2s2/C2=1A^2 s^2/C^2=1A2s2/C2=1（無因次=rad²），所以 [Sϕ]=rad2/Hz[S_\phi]=\text{rad}^2/\text{Hz}[Sϕ​]=rad2/Hz ✓。

這條與 [P1] 招牌式 Eq.(21), p.185 同一個 scaling：

L{Δω}=10log⁡10 ⁣(Γrms2qmax2⋅in2‾/Δf4 Δω2)\mathcal{L}\{\Delta\omega\}=10\log_{10}\!\left(\frac{\Gamma_{rms}^2}{q_{max}^2}\cdot\frac{\overline{i_n^2}/\Delta f}{4\,\Delta\omega^2}\right)L{Δω}=10log10​(qmax2​Γrms2​​⋅4Δω2in2​​/Δf​)

factor-of-2 教學註記（務必看）：時域「白噪 × ISF → 積分」乾淨推導得到 Sϕ=Γrms2Si/(qmax2(2πf)2)S_\phi=\Gamma_{rms}^2 S_i/(q_{max}^2(2\pi f)^2)Sϕ​=Γrms2​Si​/(qmax2​(2πf)2)，對應 SSB L=Γrms2Si/(2qmax2Δω2)\mathcal{L}=\Gamma_{rms}^2 S_i/(2q_{max}^2\Delta\omega^2)L=Γrms2​Si​/(2qmax2​Δω2)（分母 222）； 而 [P1] Eq.(21) 寫成分母 4Δω24\Delta\omega^24Δω2。差的這個 2 倍來自 SSB（單邊帶）記帳慣例 ——是文獻上著名的小爭議。它不影響 Γrms2/qmax2\Gamma_{rms}^2/q_{max}^2Γrms2​/qmax2​ 的 scaling， 也不影響 −20-20−20 dB/decade 斜率，那才是物理。本 lab 的模擬用的是乾淨時域版（分母 (2πf)2(2\pi f)^2(2πf)2）， 所以模擬曲線會貼著時域理論線。詳見 white_noise_to_phase_noise。

4. Python 核心 code​

以下逐字摘自 simulations/lab_06_white_noise_phase_noise.py 的 main()：三步驟 （生白噪 → ISF 加權 → cumsum 積分），再用 Welch 法估相位 PSD，疊上理論線。

f0 = 1.0
fs = 256.0                 # 256 samples per period
n = 2 ** 20                # ~4096 periods -> good low-freq resolution
t = np.arange(n) / fs

q_max = 1.0
S_i = 1.0e-4               # one-sided white current PSD [A^2/Hz] (normalized)

# ISF and its rms
theta_grid = np.linspace(0, 2 * np.pi, 4000, endpoint=True)
Grms = gamma_rms(theta_grid, gamma_lc_ideal(theta_grid))  # = 1/sqrt(2)

# 1) white current noise
i_n = white_noise(n, psd=S_i, fs=fs, rng=RNG)
# 2) ISF weighting
g = gamma_lc_ideal(2 * np.pi * f0 * t) * i_n / q_max
# 3) integrate (cumulative) -> excess phase
dt = 1.0 / fs
phi = np.cumsum(g) * dt
phi = phi - np.mean(phi)   # remove the random-walk DC offset for PSD est.

# 4) estimate phase PSD
f, Sphi = estimate_psd(phi, fs, nperseg=2 ** 16)

# theory line
Sphi_theory = Grms ** 2 * S_i / (q_max ** 2 * (2 * np.pi * f) ** 2)
  • 第 3 步 np.cumsum(g) * dt 就是把卷積式裡的 ∫dτ\int d\tau∫dτ 數值化——累積和就是積分器， 也就是相位「無限記憶」的來源。
  • 用的是理想 LC 的 ISF gamma_lc_ideal（Γ(θ)=−sin⁡θ\Gamma(\theta)=-\sin\thetaΓ(θ)=−sinθ），其 Γrms=1/2≈0.707\Gamma_{rms}=1/\sqrt2\approx0.707Γrms​=1/2​≈0.707（因為 ⟨sin⁡2⟩=1/2\langle\sin^2\rangle=1/2⟨sin2⟩=1/2）。

9. 如何解讀圖​

  • 藍線（simulated SϕS_\phiSϕ​）：實際從模擬相位序列估出的 PSD。它在 log–log 上是一條 斜率 −2-2−2 的直線（每十倍頻下降 202020 dB），帶有 Welch 估計的隨機起伏。
  • 黑虛線（theory）：Γrms2Si/(qmax2(2πf)2)\Gamma_{rms}^2 S_i/(q_{max}^2(2\pi f)^2)Γrms2​Si​/(qmax2​(2πf)2) 的解析線。藍線緊貼黑線 ——這就是「白噪 → 1/f21/f^21/f2」被數值證實。
  • 紅點線（−20-20−20 dB/dec guide）：純斜率參考線，確認斜率正是 −2-2−2。
  • 重點：輸入是平的（SiS_iSi​ 與 fff 無關），輸出卻是 1/f21/f^21/f2。多出來的 1/f21/f^21/f2 完全來自第 3 步那個積分器。把 ISF 換成別的形狀，只會改變 Γrms\Gamma_{rms}Γrms​（上下平移整條線）， 斜率永遠是 −2-2−2。整條線的高度由 Γrms2/qmax2\Gamma_{rms}^2/q_{max}^2Γrms2​/qmax2​ 決定，所以放大 qmaxq_{max}qmax​（tank swing）是降低 1/f21/f^21/f2 相位雜訊的主要旋鈕（設計運用見 tank_swing）。

代入本 lab 數字做一個 sanity check：Γrms2=0.5\Gamma_{rms}^2=0.5Γrms2​=0.5、Si=10−4S_i=10^{-4}Si​=10−4、qmax=1q_{max}=1qmax​=1， 在 f=0.1f=0.1f=0.1（normalized）時，2πf=0.62832\pi f=0.62832πf=0.6283，(2πf)2=0.3948(2\pi f)^2=0.3948(2πf)2=0.3948， Sϕ=0.5×10−4/0.3948=1.27×10−4S_\phi=0.5\times10^{-4}/0.3948=1.27\times10^{-4}Sϕ​=0.5×10−4/0.3948=1.27×10−4 rad²/Hz。圖上 f=0.1f=0.1f=0.1 處的高度應在此量級。

  • 理論線來源：時域乾淨版 Sϕ=Γrms2Si/(qmax2(2πf)2)S_\phi=\Gamma_{rms}^2 S_i/(q_{max}^2(2\pi f)^2)Sϕ​=Γrms2​Si​/(qmax2​(2πf)2)，對應 [P1] Eq.(21), p.185 的 scaling（差 factor-of-2，見上方註記）。

11. 限制與 approximation​

  • 這是 pedagogical toy model，非 transistor-level：ISF 用解析的 −sin⁡θ-\sin\theta−sinθ（理想 LC）， 不是從真實電路萃取。qmax=1q_{max}=1qmax​=1、f0=1f_0=1f0​=1 為 normalized 單位，沒有絕對 dBc/Hz （絕對值請見 lab_08 與 numerical_feeling 例 B）。
  • 單一白噪源、stationary（穩態）假設：真實電路有多個源、且為 cyclostationary （週期穩態，雜訊強度隨工作點週期變化），要用 Γeff=Γ⋅α\Gamma_{eff}=\Gamma\cdot\alphaΓeff​=Γ⋅α 修正 （見 effective_isf）。
  • 小角近似：把相位當成可線性疊加且為小量；大相位偏移時 L≈12Sϕ\mathcal{L}\approx\frac12 S_\phiL≈21​Sϕ​ 失準。
  • factor-of-2：模擬與時域理論線一致（分母 (2πf)2(2\pi f)^2(2πf)2），與 [P1] Eq.(21) 的 4Δω24\Delta\omega^24Δω2 差一個 SSB 記帳的 2 倍，不影響斜率與 scaling。
  • 數值限制：低頻段受總模擬時間（≈4096\approx4096≈4096 週期）與 nperseg 限制，最左端少數點 統計起伏較大；圖上只畫 f>0.02f>0.02f>0.02 的可信區段。

重點回顧​

  • 白噪（平）→ ISF 加權（縮 Γrms2/qmax2\Gamma_{rms}^2/q_{max}^2Γrms2​/qmax2​）→ 積分器（乘 1/(2πf)21/(2\pi f)^21/(2πf)2）→ 1/f21/f^21/f2 相位雜訊。
  • −20-20−20 dB/decade 斜率來自積分器，與噪聲源頻譜形狀無關。
  • 模擬 PSD 緊貼理論線 Γrms2Si/(qmax2(2πf)2)\Gamma_{rms}^2 S_i/(q_{max}^2(2\pi f)^2)Γrms2​Si​/(qmax2​(2πf)2)。
  • factor-of-2 是 SSB 記帳慣例差異，不改 scaling/斜率。
