PLL 完整相位雜訊預算與最佳 loop BW

先備：white_noise_to_phase_noise（VCO 那一項 Svco∝Γrms2/qmax2⋅Si/f2S_{vco}\propto\Gamma_{rms}^2/q_{max}^2\cdot S_i/f^2Svco​∝Γrms2​/qmax2​⋅Si​/f2 從哪來）、serdes_clocking_connection（CDR/PLL 對 VCO 的 high-pass、jitter 積分頻寬）、lc_vs_ring（為何 ring 的 SvcoS_{vco}Svco​ 高、LC 低）｜ 接下來：exercises、lab_13_pll_cdr_transfer

這頁回答一個系統設計工程師每天都要面對的問題：一顆鎖相環（PLL，phase-locked loop，把振盪器相位鎖到參考時鐘的負回授環）的輸出相位雜訊，到底是由哪些源、各自貢獻 多少、在哪個 offset 頻段誰當家？而 loop bandwidth（環路頻寬，回授追得上的最高 offset） 應該選多寬，才能讓總抖動最小？ 我們要把五個雜訊源逐一寫出它們到輸出的轉移函數，加總成

Sout=(SrefN2+Scp) ∣Hlp∣2+Svco ∣Hhp∣2S_{out}=(S_{ref}N^2+S_{cp})\,\lvert H_{lp}\rvert^2+S_{vco}\,\lvert H_{hp}\rvert^2Sout​=(Sref​N2+Scp​)∣Hlp​∣2+Svco​∣Hhp​∣2

（規範第 11.2 節「PLL 輸出雜訊預算」），再對 ∫Sout df\int S_{out}\,df∫Sout​df（積分相位變異，正比於 rms jitter 平方）求極小，得到那條著名的 U 形曲線與其最低點。

物理直覺（先講結論）：PLL 是一個低通追蹤器。在 loop bandwidth fnf_nfn​ 以內，回授來得及 反應，輸出跟著參考走——於是 reference 與環路前端（PFD、charge-pump、divider）的雜訊 被放大且低通地搬到輸出（而且 reference 還被 ×N\times N×N 倍頻，功率 ×N2\times N^2×N2）；同時 VCO 自己的 close-in 漂移被回授糾正掉（VCO 高通）。在 fnf_nfn​ 以外，回授來不及，輸出跟著 VCO 自由跑——VCO 的 1/f21/f^21/f2 雜訊原樣漏出。所以 in-band 跟 ref/CP、out-of-band 跟 VCO， 交越就在 fnf_nfn​。把 fnf_nfn​ 開太窄→VCO 漏出太多（U 形左臂上揚）；開太寬→ref/CP 被搬出來太多 （右臂上揚）。中間必有一個最佳 fnf_nfn​。

本頁的 PLL 閉環轉移函數（loop transfer / 開環增益 / type-II 穩定性那些高階細節）屬於 標準 PLL 文獻（Gardner、Razavi、Best），不在本站下載的 5 篇 PDF 之內；我們只引用其 type-II 二階閉環結果（規範第 10.2 節已收錄），重心放在「ISF 決定 VCO 那一項 SvcoS_{vco}Svco​」 以及「預算如何加總、最佳 BW 如何取」。VCO 那一項的微觀來源（Γrms2/qmax2\Gamma_{rms}^2/q_{max}^2Γrms2​/qmax2​）正是 本站前面整套 ISF 理論的成果。

為什麼要做「雜訊預算」​

phase noise 不是單一數字，它是一條隨 offset 變化的曲線；而曲線上每一段由不同的源主宰。 做預算（budget）= 把每個源畫成一條，看誰在哪段冒頭、總和長什麼樣。 這件事的價值：

  • 找瓶頸：close-in 太高？多半是 reference 或 charge-pump（被 N2N^2N2 放大）。far-out 太高？ 是 VCO。對症下藥，不要盲目換零件。
  • 選 loop BW：交越點、總 jitter 都跟 fnf_nfn​ 強相關，預算讓你量化這個取捨。
  • 連到系統指標：把 SoutS_{out}Sout​ 積分得 rms jitter σt\sigma_tσt​，直接餵進 SerDes 的 eye/BER（見 serdes_clocking_connection）。

第 2 步：加總成輸出預算​

reference 與 charge-pump/divider 走同一條低通路徑（reference 先 ×N\times N×N），VCO 走高通路徑， 三段不相關、功率相加（規範第 11.2 節）：

Sout(f)=(Sref(f) N2+Scp(f)) ∣Hlp(f)∣2+Svco(f) ∣Hhp(f)∣2.S_{out}(f)=\big(S_{ref}(f)\,N^2+S_{cp}(f)\big)\,\lvert H_{lp}(f)\rvert^2+S_{vco}(f)\,\lvert H_{hp}(f)\rvert^2 .Sout​(f)=(Sref​(f)N2+Scp​(f))∣Hlp​(f)∣2+Svco​(f)∣Hhp​(f)∣2.

  • divider 去哪了：SdivS_{div}Sdiv​ 與 charge-pump 走同一條低通路徑、在輸出同樣是 ∣Hlp∣2\lvert H_{lp}\rvert^2∣Hlp​∣2 整形，故工程上常把 SdivS_{div}Sdiv​ 併進 ScpS_{cp}Scp​ 當作「環路前端等效 in-band 地板」。本頁的 ScpS_{cp}Scp​ 就是「PFD + charge-pump + divider」的合計。

VCO 那一項就是本站的 ISF 結果​

第 4 步：reference spur（簡述）​

除了隨機相位雜訊（連續的裙邊），PLL 輸出還常有離散的 spur（雜散，單一頻率的尖刺）。 最常見的是 reference spur：charge-pump 在每個參考週期注入電流脈衝，這個週期性擾動以 freff_{ref}fref​ 的整數倍（即 offset =±fref,±2fref,…=\pm f_{ref},\pm2f_{ref},\dots=±fref​,±2fref​,…）出現在輸出。來源是 CP current mismatch、leakage、PFD dead-zone，把控制電壓調出一個 freff_{ref}fref​ 的小漣波，再被 VCO 的 KvK_vKv​ 轉成相位調制邊帶。

  • spur vs 隨機 PN：spur 是確定性、窄的線（在頻譜上是一根針），隨機 PN 是連續裙邊； 量測上 spur 不隨解析頻寬 RBW 改變高度（功率集中在一根），隨機 PN 的 dBc/Hz 才是「per-Hz」。
  • 與 loop BW 的關係：reference spur 在 offset freff_{ref}fref​ 處，若 fref>fnf_{ref}>f_nfref​>fn​ 會被 ∣Hlp∣2\lvert H_{lp}\rvert^2∣Hlp​∣2 低通衰減（loop BW 越窄、spur 越被壓）；這與「窄 BW 對隨機 in-band 有利」一致，但會犧牲 VCO 抑制——又是同一個取捨。
  • 本頁的預算只算隨機部分（連續 SoutS_{out}Sout​）；spur 的定量分析屬標準 PLL 文獻（不在 5 篇 PDF 內），這裡僅作概念連結。

  • 第一塊（ref/CP）隨 fnf_nfn​ 單調增：fnf_nfn​ 越大，低通通帶越寬，把越多 in-band 地板（含被 N2N^2N2 放大的參考）搬到輸出。粗估 ∝(SrefN2+Scp)⋅fn\propto(S_{ref}N^2+S_{cp})\cdot f_n∝(Sref​N2+Scp​)⋅fn​（平坦地板乘以通帶寬度）。
  • 第二塊（VCO）隨 fnf_nfn​ 單調減：fnf_nfn​ 越大，高通把越多 VCO 的 1/f21/f^21/f2 close-in 糾正掉。對 Svco=k/f2S_{vco}=k/f^2Svco​=k/f2 經高通，殘留積分 ∝k/fn\propto k/f_n∝k/fn​（BW 越寬、漏出越少）。

一增一減 → U 形。對 fnf_nfn​ 求導令為零，存在唯一極小：

d σϕ2dfn=0⟹（ref/CP 漏出的邊際增加）=（VCO 壓制的邊際減少）.\frac{d\,\sigma_\phi^2}{d f_n}=0\quad\Longrightarrow\quad \text{（ref/CP 漏出的邊際增加）}=\text{（VCO 壓制的邊際減少）}.dfn​dσϕ2​​=0⟹（ref/CP 漏出的邊際增加）=（VCO 壓制的邊際減少）.

用上面兩個粗估（a fn+b/fna\,f_n+b/f_nafn​+b/fn​ 形式，a∝SrefN2+Scpa\propto S_{ref}N^2+S_{cp}a∝Sref​N2+Scp​、b∝Svcob\propto S_{vco}b∝Svco​ 係數）求極小：

ddfn ⁣(afn+bfn)=a−bfn2=0 ⟹ fn\*=ba ∝ Svco 係數SrefN2+Scp.\frac{d}{df_n}\!\left(a f_n+\frac{b}{f_n}\right)=a-\frac{b}{f_n^2}=0\ \Longrightarrow\ f_n^\*=\sqrt{\frac{b}{a}}\ \propto\ \sqrt{\frac{S_{vco}\text{ 係數}}{S_{ref}N^2+S_{cp}}} .dfn​d​(afn​+fn​b​)=a−fn2​b​=0 ⟹ fn\*​=ab​​ ∝ Sref​N2+Scp​Svco​ 係數​​.
  • 物理意義：VCO 越吵（bbb 大）→ 最佳 BW 越大（要更寬的環去壓 VCO）；ref/CP 越吵或 NNN 越大（aaa 大）→ 最佳 BW 越小（不能把太多 in-band 地板搬出來）。這條 fn\*∝b/af_n^\*\propto\sqrt{b/a}fn\*​∝b/a​ 是 PLL 設計的核心直覺，雖然係數要靠數值積分定。
  • toy 註記：上面 afn+b/fnaf_n+b/f_nafn​+b/fn​ 是把整形近似成理想磚牆濾波的啟發式估算；真實積分要用完整 ∣H∣2\lvert H\rvert^2∣H∣2（含 fnf_nfn​ 附近的 peaking），故下面用 lab_20 的數值積分給精確最低點。

核心 Python（完整 script：simulations/lab_20_pll_budget.py）：

import numpy as np
from simulations.common.pll_utils import H_lowpass_mag2, H_highpass_mag2

def output_psd(f, fn, N, zeta=0.707):
    lp = H_lowpass_mag2(f, fn, zeta)
    hp = H_highpass_mag2(f, fn, zeta)
    S_ref = 1e-16 + 1e-18 * (1e6 / f)   # 乾淨晶體
    S_cp  = 5e-13 * np.ones_like(f)     # PFD/CP/divider 平坦底
    S_vco = 2e-10 * (1e6 / f) ** 2      # ring VCO -100 dBc/Hz @1MHz, 1/f^2
    return (S_ref * N**2 + S_cp) * lp + S_vco * hp   # 預算加總

f = np.logspace(3, 9, 3000); f0 = 5e9; N = 100
fns = np.logspace(4.5, 7.5, 60)
jit = [np.sqrt(np.trapezoid(output_psd(f, fn, N), f)) / (2*np.pi*f0) for fn in fns]
k = int(np.argmin(jit))
print(fns[k]/1e6, "MHz", jit[k]*1e15, "fs")   # -> ~6.90 MHz, ~259 fs

與 SerDes 的關聯​

PLL 輸出的 σt\sigma_tσt​（本頁右圖最低點 ≈259\approx259≈259 fs）就是餵給高速串列收發器 （SerDes）取樣時脈的抖動預算。在 serdes_clocking_connection 裡，這個 σt\sigma_tσt​ 直接決定眼圖（eye diagram）的水平閉合與 BER（bit error rate）：UI（unit interval，單位間隔）越短（資料率越高），同樣的 σt\sigma_tσt​ 吃掉的眼寬比例越大。所以選對 loop BW 把 PLL jitter 壓到最低，是整條 SerDes link 預算的源頭。CDR（時脈資料回復）本身也是 一個 PLL，它對輸入 jitter 的 jitter-tolerance（容忍）轉移就是這裡的 ∣Hlp∣2\lvert H_{lp}\rvert^2∣Hlp​∣2 （低頻 jitter 追得上→容忍、高頻→靠眼圖裕度），見 lab_13_pll_cdr_transfer。

重點回顧​

  • PLL 輸出預算：Sout=(SrefN2+Scp)∣Hlp∣2+Svco∣Hhp∣2S_{out}=(S_{ref}N^2+S_{cp})\lvert H_{lp}\rvert^2+S_{vco}\lvert H_{hp}\rvert^2Sout​=(Sref​N2+Scp​)∣Hlp​∣2+Svco​∣Hhp​∣2（規範 11.2）。
  • in-band 跟 ref/CP（被 N2N^2N2 放大、低通）、out-of-band 跟 VCO（高通、1/f21/f^21/f2 漏出），交越在 fnf_nfn​。
  • VCO 那一項就是本站 ISF 結果 Svco∝Γrms2/qmax2⋅Si/f2S_{vco}\propto\Gamma_{rms}^2/q_{max}^2\cdot S_i/f^2Svco​∝Γrms2​/qmax2​⋅Si​/f2。
  • reference spur 是離散尖刺（CP mismatch/leakage 的 freff_{ref}fref​ 漣波），窄 BW 可壓但犧牲 VCO 抑制。
  • 最佳 loop BW：對 ∫Soutdf\int S_{out}df∫Sout​df 求極小，fn\*∝Svco/(SrefN2+Scp)f_n^\*\propto\sqrt{S_{vco}/(S_{ref}N^2+S_{cp})}fn\*​∝Svco​/(Sref​N2+Scp​)​； 太窄 VCO 漏出、太寬 ref/CP 漏出。lab_20 數值：fn\*≈6.90f_n^\*\approx6.90fn\*​≈6.90 MHz、σt≈259\sigma_t\approx259σt​≈259 fs。
  • 這顆 ring-PLL U 形左臂比右臂陡 → 偏好稍大的 loop BW。
